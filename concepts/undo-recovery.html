Core Git 7/8

Undo And Recovery

Recover fast without damaging shared history. Choose the command based on what layer you need to fix: use restore for file-level undo, revert for shared history-safe rollback, and reset only when rewriting private local history is acceptable.

Back to Git Concepts overview

Recovery Order Of Preference

restore file changes revert shared commit reset private branch reflog for rescue FlexDeploy version restore

Restore: File-Level Undo

No History Rewrite
git restore path/to/file
git restore --staged path/to/file

Reset vs Revert On Commit History

Shared Safety
git reset --soft <sha> (local/private only)
git revert <sha> (recommended on shared branches)

Two Rollback Options: Deploy State vs Git History

Operational + Code Recovery

Fast environment recovery can be FlexDeploy version restore. Git history correction should still be PR-governed.

git switch -c revert/fix-issue
git revert <sha> || apply forward fix
git push -u origin revert/fix-issue

Reflog Rescue For Lost Commits

Last Resort Recovery
git reflog
git checkout -b recovery HEAD@{1}
